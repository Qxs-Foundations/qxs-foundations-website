Git Infrastructure

Independent source control

We plan to operate a self-hosted Git collaboration portal to reduce reliance on third-party SaaS hosting and safeguard codebase integrity.

Self-Hosted UI

Forgejo or Gitea

Our future Git infrastructure (git.qxs-foundations.org.in) will utilize highly auditable self-hosted engines like Forgejo or Gitea, returning code custody to the community.

Code Repositories

Project repositories

All core libraries, compiler binaries, and OS drafts will reside on fully sovereign drives, isolating tracking systems from third-party account suspensions.

Open Governance

Community transparency

Public repository dashboards will host discussions, project wiki indexes, and community review streams, ensuring open contribution channels.

Mirroring

Migration strategy

We will mirror our key codebases to secondary outlets like GitHub for visibility while keeping the absolute, cryptographically signed primary repository on sovereign drives.

Autonomy by Design

Protecting the roots of open source

Hosting open-source code on massive, corporate-owned proprietary clouds introduces quiet architectural compromises. Arbitrary service shifts, automated account bans, and aggressive dataset training models undermine true builder freedom.

By operating our own Git portal on sovereign networks, QXS Foundations ensures that the very roots of our applications, compilers, and security tools remain free and immune to centralized platform filters.
